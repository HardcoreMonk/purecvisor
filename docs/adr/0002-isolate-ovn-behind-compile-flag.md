# ADR-0002: OVN/SDN 모듈을 컴파일 플래그 뒤로 격리한다

날짜: 2026-04-10
상태: accepted

## 맥락
DESIGN.md §0은 "중소 규모 3~16노드 사설 KVM 클러스터"를 대상으로 명시하며,
§6은 OVN 수준의 SDN을 명시적 비범위로 두지 않았으나 사실상 운영 시나리오
S1~S13 어디에도 매핑되지 않는다.

코드 감사 결과 (2026-04-10) OVN 관련 RPC가 18개 등록되어 있다:
ovn.{acl, dhcp, nat, port, router, switch, tenant} 계열. 어떤 시나리오도
이들을 요구하지 않으며, S1(VM 대량 생성)·S2(마이그레이션)는 기본 Linux
브릿지 + dnsmasq로 충분함이 3노드 클러스터 운영으로 검증됨.

OVN을 유지할 때의 비용:
- 빌드 의존: ovn-common, ovn-host (클러스터 빌드 그래프 확장)
- 테스트 부채: OVN 통합 테스트 14케이스 유지 비용
- 보안 표면: OVSDB 소켓, OVN northbound/southbound 추가 노출
- 코드량: ~6,569 LOC 중 OVN 비중이 가장 큼 (network 디렉터리)
- 인지 부하: 신규 기여자가 "이게 왜 있지?"를 매번 묻게 됨

OVN을 즉시 삭제할 때의 위험:
- 일부 운영자가 이미 OVN 기능을 켜고 운영 중일 가능성 (감사 로그 미수집)
- 클러스터 사용자가 페더레이션과 함께 쓰고 있을 가능성
- 일괄 삭제는 회고에서 경고한 "리팩토링 Phase 1~4" 안티패턴의 재현

## 결정
OVN/SDN을 즉시 삭제하지 않는다. 대신 단일/클러스터 빌드가 사용한 동일 패턴
으로 컴파일 플래그 뒤로 격리한다:

1. `-DPCV_OVN_ENABLED=0/1` 도입, **기본값 0** (off)
2. 빌드 타깃 분기:
   - `make multi` (기본): PCV_OVN_ENABLED=0
   - `make multi-ovn`: PCV_OVN_ENABLED=1 (선택적, deprecated 표시)
3. dispatcher.c에서 ovn.* 핸들러 등록을 `#if PCV_OVN_ENABLED` 가드
4. PCV_OVN_ENABLED=0 빌드에서 ovn.* 호출 시 -32601 Method not found 반환
5. CI에 "PCV_OVN_ENABLED=0 빌드에 ovn 심볼 부재" 검증 잡 추가
   (Single Edge `cluster` 심볼 부재 검증과 동일 패턴)
6. **6개월 후 (2026-10-10) 재평가**: 그동안 `--enable-ovn` 사용 보고가 0건
   이면 코드 자체를 삭제. 보고가 있으면 해당 시나리오를 DESIGN.md S14로
   추가하고 본 ADR을 superseded 처리.

## 결과
- 좋음: 기본 빌드의 RPC 표면이 18개 줄어듦 (~250 → ~232)
- 좋음: 빌드 의존 ovn-common 제거, 클러스터 바이너리 크기 감소 예상
- 좋음: 보안 표면 축소 (OVSDB 소켓 미노출)
- 좋음: 신규 기여자 인지 부하 감소
- 좋음: "삭제" 결정을 6개월 미루므로 기존 사용자 충격 0
- 나쁨: 컴파일 플래그가 또 하나 늘어남 (Single/Multi에 이어 OVN까지)
- 나쁨: 6개월간 코드는 그대로 남아 유지보수 비용 지속
- 포기한 것: "즉시 삭제"의 단순함. 회고 교훈에 따라 점진적 격리 선택.

## 하지 않기로 한 것
- **OVN을 다시 기본값으로 켜는 것**: 어떤 운영 시나리오도 OVN을 요구하지
  않는다. "있으니까 켠다"는 정당화가 아니다. 켜려면 DESIGN.md에 시나리오
  먼저 추가.
- **OVN을 별도 리포로 분리**: 단일/클러스터 분리는 1년의 비용을 만들었
  다. 같은 실수를 반복하지 않는다. 컴파일 플래그로 충분.
- **OVN과 OVS를 함께 격리**: OVS(Open vSwitch)는 S2 마이그레이션의 VXLAN
  오버레이에 사용 중이며 S1/S2에서 정당화된다. OVN(상위 SDN 컨트롤러)만
  격리한다.
- **DPDK/SR-IOV/GPU/Federation을 본 ADR에 묶기**: 각자 별도 ADR로 다룬다
  (ADR-0003~). 한 번에 묶어 처리하는 것이 회고가 경고한 안티패턴.
