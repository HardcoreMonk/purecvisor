# ADR-0006: GPU/iSCSI 타겟/Federation을 시나리오 S14~S16으로 흡수한다

날짜: 2026-04-10
상태: accepted (supersedes ADR-0004)

## 맥락
ADR-0004는 GPU/iSCSI 타겟/Federation 3개 카테고리를 deprecated 표시 + 컴파
일 플래그 격리 + 3개월 후 삭제 결정 경로로 두었다. 그러나 본 ADR-0004
작성 직후, 운영자(작성자) 판단으로 다음을 확인:

- **GPU**: 향후 AI/ML 추론 워크로드 수용 가능성이 실재. 정적 할당만 지원
  하더라도 vfio-pci/mdev 인프라는 현재 코드 수준에서 검증되어 있다.
- **iSCSI 타겟**: ZFS 풀의 직접 가치 활용. 외부 호스트가 PureCVisor를
  블록 스토리지로 사용하는 사용 패턴이 §0의 "사설 KVM 클러스터"와 모순
  되지 않으며 오히려 ZFS 자산을 다용도화한다.
- **Federation**: §6의 "사이트 자동 페일오버 제외"는 유지하되, *조회 +
  수동 마이그레이션*에 한정한 페더레이션은 DR 사이트 운영자의 실제 요구.

ADR-0004의 deprecated 결정은 "DESIGN.md 시나리오 미정의"가 단일 근거였다.
정의를 추가하면 부채가 자산이 된다. 회고 교훈("기능 명세 권한을 운영에
넘긴다")의 정직한 적용은 *삭제*가 아니라 *시나리오로 박제*다. 단, 박제는
범위를 동시에 명시적으로 좁혀야 한다 — 그렇지 않으면 다시 사후 패치 사이
클로 회귀.

## 결정
ADR-0004를 supersede하고 다음을 적용:

1. DESIGN.md에 시나리오 3건 신설:
   - **S14 GPU 패스스루/vGPU** — 정적 할당만, 스케줄링/풀 공유 제외
   - **S15 iSCSI 타겟 노출** — CHAP 필수, NFS/SMB/S3 제외
   - **S16 멀티사이트 페더레이션** — 조회 + 수동 마이그레이션, 자동 페일
     오버 제외
2. DESIGN.md §0 "부가 워크로드" 단락 신설 (S14~S16, 선택적)
3. DESIGN.md §5 매핑표에 3행 추가
4. DESIGN.md §6 "Won't Do"에 각 시나리오의 명시적 비범위 3줄 추가:
   - GPU 스케줄링 / time-slicing / k8s device plugin
   - NFS/SMB/S3 게이트웨이
   - 사이트 자동 페일오버 / 글로벌 단일 네임스페이스
5. ADR-0004의 deprecated 표시/3개월 윈도우는 무효화. 컴파일 플래그
   격리도 도입하지 않음 — 시나리오로 정당화되었으므로 기본 빌드에 포함.
6. 각 카테고리는 S14~S16의 통과 조건을 만족하는 통합 테스트가 6개월 안에
   추가되어야 한다. 미충족 시 본 ADR을 다시 supersede하여 격리 결정 가능.

## 결과
- 좋음: 부채 ~10건이 자산으로 전환 (매핑됨 ~142 → ~152)
- 좋음: 컴파일 플래그 3개 추가 부담 회피
- 좋음: §6에 시나리오별 비범위가 3줄 더 명시되어 향후 스코프 크리프 방지
- 좋음: §0이 "선택적 부가 워크로드" 개념을 갖게 됨 — 향후 유사 결정의
  템플릿
- 나쁨: §0의 제품 정의가 약간 확장됨 ("순수 KVM 오케스트레이터" → "+
  선택적 부가 워크로드"). 정체성이 흐려질 위험.
- 나쁨: 통합 테스트 부채 신설 (6개월 내 작성 의무)
- 포기한 것: ADR-0004의 단순한 "삭제" 경로. 흡수는 명시적 범위 한정의
  지속적 규율을 요구한다.

## 하지 않기로 한 것
- **S14~S16의 범위 확장**. 각 시나리오의 "하지 않을 것" 단락은 미래의
  기능 추가 요청에 대한 1차 거부 근거다. 이를 우회하려면 본 ADR을
  supersede.
- **성공 사례를 다른 회색지대(ADR-0005)로 일반화**. ADR-0005는 데이터 기반
  결정 경로로 그대로 유지. "흡수가 가능하다"가 "모든 것을 흡수해야 한다"
  로 미끄러지면 안 된다.
- **선택적 활성/비활성 토글을 다시 도입**. 시나리오로 정당화된 기능은
  기본 빌드에 포함된다. 끄고 싶으면 호출하지 않으면 된다.
- **§0을 "범용 가상화 플랫폼"으로 재정의**. 본 ADR은 부가 워크로드 3개를
  더한 것이지 정체성 변경이 아니다.
